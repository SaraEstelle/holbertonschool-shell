Holberton School – Shell Projects
System Engineering & DevOps – Foundations

This repository contains all Shell projects completed as part of the Holberton School curriculum.
These projects introduce the fundamentals of working in a Unix/Linux environment, using the command line, manipulating files, managing permissions, redirecting input/output, and customizing the shell environment.

The Shell track is essential for any aspiring software engineer, DevOps engineer, or system administrator, as it builds the foundation for automation, scripting, and understanding how operating systems work under the hood.

🎯 Learning Goals of the Shell Track
Across all projects in this repository, you will learn to:

Navigate and manipulate the Linux filesystem

Understand and use shell commands efficiently

Work with file permissions and user/group management

Redirect input and output streams

Use pipes and filters to process data

Understand shell initialization files

Work with environment variables and expansions

Create aliases and customize your shell

Write simple shell scripts following strict formatting rules

These skills form the backbone of command‑line proficiency and DevOps practices.

📘 Project Summaries
1. Shell, Basics
Learn how to:

Navigate the filesystem (cd, pwd, ls)

Understand directories (., .., /, ~)

Work with files and links

Use basic commands (cp, mv, rm, mkdir)

Read man pages

Understand the difference between shell and terminal

This project builds the foundation for all future shell work.

2. Shell, Permissions
Learn how to:

Modify file permissions (chmod)

Change ownership (chown, chgrp)

Switch users (su, sudo)

Understand user IDs and groups

Create users and groups

Work with symbolic and hard links

This project teaches the Linux security model and access control.

3. Shell, I/O Redirections and Filters
Learn how to:

Redirect input/output (>, >>, <)

Use pipes (|)

Filter and transform text (grep, sort, uniq, tr, cut, rev)

Read specific parts of files (head, tail)

Work with /etc/passwd and /etc/shadow

Combine commands to build powerful one‑liners

This project develops your ability to process data efficiently from the command line.

4. Shell, Init Files, Variables and Expansions
Learn how to:

Understand shell startup files (/etc/profile, ~/.bashrc)

Work with local and global variables

Use reserved variables (HOME, PATH, PS1)

Perform shell arithmetic

Use expansions and command substitution

Create and manage aliases

Execute commands from files (source, .)

This project teaches how the shell works internally and how to customize your environment.

🧪 Requirements Across All Projects
Scripts must run on Ubuntu 20.04 or 22.04 LTS

Allowed editors: vi, vim, emacs

All scripts must be exactly 2 lines long (unless specified otherwise)

First line must always be:

bash
#!/bin/bash
No use of backticks, &&, ||, or ;

All scripts must be executable

Each project folder must contain its own README.md

🏁 Conclusion
This repository represents the complete Shell foundation of the Holberton curriculum.
By completing these projects, you gain the essential skills needed to:

Work confidently in a Linux environment

Automate tasks

Understand system behavior

Prepare for more advanced DevOps, scripting, and backend engineering topics
